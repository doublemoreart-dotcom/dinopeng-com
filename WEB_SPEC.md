# 網頁閱讀性與更新規範

## 目的

本規範適用於所有 `ai_industry_penetration_YYYY-MM-DD.html` 週更版本。每次更新資料時，除了更新數據與來源，也必須維持閱讀性、主題切換、字級切換與 RWD。

## 閱讀性與留白

- 頁面主內容必須置中，最大寬度建議落在 `960px` 到 `1120px`。
- 上下左右 Margin / Padding 需保留適度留白，不讓文字或圖表貼齊視窗邊緣。
- 桌面版左右留白需比手機版更寬；手機版仍需保留至少 `16px` 左右內距。
- 區塊之間使用一致間距，避免卡片、圖表與說明文字擠在一起。
- 圖表高度需固定或以 RWD 條件調整，避免載入後版面跳動。

## Light / Dark Mode

- 每個週更 HTML 都必須提供 Light / Dark Mode 切換。
- 預設可跟隨系統或使用上次選擇；若使用 localStorage，需用 `try/catch` 避免瀏覽器權限錯誤。
- Light 與 Dark 都必須維持文字、卡片、邊框與圖表背景的對比。
- 切換時需更新 `aria-pressed`，讓目前選取狀態可被輔助工具辨識。

## 字級大小

- 每個週更 HTML 都必須提供三種字級：
  - 小
  - 中
  - 大
- 預設字級為「中」。
- 字級切換需影響主要文字、卡片、標籤與數值，不只影響單一段落。
- 切換時需更新 `aria-pressed`。

## RWD

- 頁面必須符合 RWD，至少支援桌面、平板與手機寬度。
- KPI 卡片在桌面可雙欄，手機需改為單欄。
- 產業長條圖在手機需保留可讀性，產業名稱、長條與百分比不能互相擠壓。
- 控制列在窄螢幕需可換行或改為單欄。
- 圖表 canvas 必須有穩定高度，手機版可降低高度但不能過窄。

## 每次週更檢查

更新 `ai_industry_penetration_YYYY-MM-DD.html` 時，需檢查：

- 是否保留 `.reader-controls` 控制列。
- 是否保留 Light / Dark Mode 按鈕。
- 是否保留三段字級按鈕。
- 是否有 `@media (max-width: ...)` RWD 規則。
- 是否保留適度頁面留白與主內容最大寬度。
- 新增文字是否在小字級與大字級下都可讀。
- 新增圖表或卡片是否在手機版不溢出。
